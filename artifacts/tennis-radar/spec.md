# spec.md — Tennis Radar

## 개요
날씨와 공공 테니스코트 예약 현황을 세 예약 시스템(gytennis 고양 / 양평누리 영등포 / yeyak 서울)에서 통합해 "지금·오늘 칠 수 있나?"를 한 화면으로 보여주고, 예약 페이지로 바로 연결한다. 소스마다 제공 세밀도가 달라(슬롯 단위 vs 날짜 단위) 카드에 **혼합 해상도**로 정직하게 표시한다.

## 범위

### 포함
- 실위치 기반 코트 목록. 위치 없으면 여의도 트윈타워 기준. 사용자가 위치 수동 지정 가능
- 반경 필터 (기본 10km, 사용자 선택 가능)
- 날짜 선택: 캘린더에서 임의 날짜 복수 선택 + 평일/주말 프리셋. 선택 날짜는 칩으로 표시
  - 각 카드는 선택 날짜별로 한 줄씩 표시하고, 슬롯 소스는 그 날짜의 가능 시간칩을 인라인으로 함께 보여준다(마감인 날은 "마감"). yeyak은 날짜별 N면 남음
- 시간대 필터(선택적): 별도 창(모달)에서 1시간 해상도로 하나 이상의 시간 구간을 선택(연속·불연속 모두). 코트별 예약 단위(1h/2h 등)가 달라 필터는 1시간 해상도로 하고, 선택 구간과 겹치는 예약가능 블록이 있는 슬롯 소스 코트를 노출. yeyak은 "시간 미상"으로 유지. 미선택 시 전체
- 핵심은 **예약가능한 코트 시간**을 보여주는 것. 날씨는 그 위의 **보조 힌트**
- **날씨 = 작은 이모지 힌트**: 예약가능 시간칩에 작은 날씨 이모지를 붙여 강수 상태를 표시. 강수확률·강수량은 단계별 이모지(☀️/🌦️/🌧️/⛈️)와 색상으로 한눈에 식별. 정확한 강수확률(%)·강수량(mm) 수치는 칩 탭/호버 시 툴팁으로 확인 가능(평상시엔 노출 안 함). 코트 좌표 기준
- 슬롯 소스(gytennis·양평누리): 예약가능한 **시간대 칩** 나열, 각 칩에 작은 날씨 이모지
- 날짜 소스(yeyak): **"N면 남음"**(잔여 = 총정원 − 예약수) + 하루 대표 날씨
- 오늘 탭에서는 현재 시각 이후 시간대만 표시
- **코트 상세 정보**: 주소, 코트 면 수, 표면(하드/클레이/인조잔디 등), 이용료, 운영시간, 문의처 — 소스에서 제공되는 범위 내에서 앱에 직접 표시
- 코트 카드 → 해당 예약 시스템 딥링크 (새 탭)
- 결과 목록: 거리순 단일 스크롤 (페이지네이션·무한스크롤 없음). 코트 모수가 작아 반경 필터 후 한 화면 규모라 충분

### 제외
- 앱 내 직접 예약/결제 — 세 시스템의 인증 체계가 달라 통합 불가
- 슬롯 지정 딥링크 — 세 시스템 모두 딥링크가 날짜/시설 단위라 특정 슬롯으로 바로 못 감
- yeyak 시간대별(회차) 현황 — 로그인 후 예약 폼에서만 노출. MVP에서 날짜 단위로만 사용
- 목동테니스장 등 로그인 필요 시스템 — 비로그인 조회 불가로 제외
- 민간코트 — 각자 시스템 달라 범위 초과
- 알림(push) — Phase 2
- 파트너 매칭 — 다른 문제
- 온도·습도 등 기타 날씨 — 플레이 판단에 불필요

## 시나리오

### 1. 최초 진입 — 실위치 허용
- **Given** — 사용자가 앱에 처음 진입, 위치 정보 없음
- **When** — 브라우저 위치 권한 요청에서 "허용" 선택
- **Then** — 현재 위치 기준 반경 10km 내 코트 목록이 거리 가까운 순으로 표시된다

성공 기준:
- [ ] 위치 허용 후 코트 카드 목록이 거리순으로 나타난다
- [ ] 각 카드에 코트명·거리(km)·오늘 날짜의 시간대별 날씨·예약 현황이 표시된다
- [ ] 반경 10km 밖 코트는 목록에 나타나지 않는다

---

### 2. 진입 — 위치 권한 거부 (여의도 fallback)
- **Given** — 사용자가 위치 권한을 거부했거나 이미 거부 상태
- **When** — 앱이 위치를 얻지 못함
- **Then** — 여의도 트윈타워 좌표를 기준으로 반경 10km 코트 목록이 표시되고, 기본 위치 사용 중임을 알리는 안내가 보인다

성공 기준:
- [ ] 위치 권한 없이도 코트 목록이 렌더된다
- [ ] "현재 위치 대신 기본 위치(여의도)를 사용 중입니다" 안내 문구가 표시된다
- [ ] 안내 옆 "현재 위치 사용하기" 버튼 클릭 시 위치 권한 재요청이 트리거된다

---

### 3. 위치 수동 지정
- **Given** — 코트 목록이 표시된 상태
- **When** — 사용자가 위치 지정 UI에서 다른 위치를 선택/입력
- **Then** — 지정한 위치 기준으로 거리·목록·날씨가 다시 계산되어 표시된다

성공 기준:
- [ ] 위치 변경 후 각 카드의 거리(km)가 새 위치 기준으로 갱신된다
- [ ] 새 위치 반경 밖 코트는 목록에서 사라지고, 반경 안 코트가 나타난다
- [ ] 현재 적용 중인 위치가 화면에 표시된다

---

### 4. 반경 조절
- **Given** — 코트 목록이 10km 반경으로 표시된 상태
- **When** — 반경을 5km로 변경
- **Then** — 5km 밖 코트가 목록에서 사라진다

성공 기준:
- [ ] 반경 변경 후 반경 밖 코트가 목록에서 제거된다
- [ ] 현재 반경 값(예: "10km")이 UI에 표시된다
- [ ] 반경 내 코트가 없으면 "현재 반경 내 코트가 없습니다. 반경을 늘려보세요" 메시지가 표시된다

---

### 5. 날짜 복수 선택
- **Given** — 코트 목록이 표시되고 캘린더에서 날짜를 고를 수 있는 상태
- **When** — 캘린더에서 7/22, 7/25 두 날짜를 선택 (또는 "평일"·"주말" 프리셋 선택)
- **Then** — 선택 날짜가 칩으로 표시되고, 각 카드가 선택 날짜별 스트립으로 예약 현황·날씨를 보여준다

성공 기준:
- [ ] 선택한 날짜가 칩(예: "7/22", "7/25")으로 표시되고 칩의 ✕로 개별 해제된다
- [ ] "평일" 프리셋 선택 시 조회 가능 범위 내 평일들이 선택된다
- [ ] 각 카드가 선택 날짜별 한 줄로 표시되고, 슬롯 소스는 그 날짜의 가능 시간칩을 인라인으로 함께 보여준다
- [ ] 슬롯 소스에서 어떤 날짜가 마감이면 칩 없이 "마감"으로 표시된다
- [ ] 슬롯 소스는 각 시간칩에 그 시간대의 강수(아이콘·강수확률·강수량)를 함께 표시한다
- [ ] yeyak 카드는 날짜별 N면 남음 + 날짜별 날씨 이모지(슬롯 소스와 동일 규칙)로 표시된다

---

### 5-1. 시간대 필터 (모달 · 1시간 해상도 · 다중 구간)
- **Given** — 코트 목록이 표시되고 시간 필터가 미선택(전체)인 상태
- **When** — "시간" 버튼으로 창을 열어 막대에서 18:00~22:00을 선택(또는 12:00~14:00과 18:00~22:00을 불연속으로 선택)하고 적용
- **Then** — 창이 닫히고, 선택 구간과 겹치는 예약가능 블록이 있는 슬롯 소스 코트만 남는다

성공 기준:
- [ ] 선택 구간에 겹치는 가능 블록이 없는 슬롯 소스 코트는 목록에서 제외된다
- [ ] 예약 단위가 1시간인 코트든 2시간인 코트든, 구간과 겹치기만 하면 노출된다
- [ ] 불연속 다중 구간(예: 12-14 + 18-22) 선택이 가능하다
- [ ] 슬롯 소스 카드는 선택 구간 내 시간대 칩을 우선 표시한다
- [ ] yeyak 코트는 시간을 특정할 수 없어 "시간 미상" 표시와 함께 목록에 유지된다
- [ ] 필터를 해제하면 모든 코트가 다시 표시된다

---

### 6. 슬롯 소스 카드 — 시간칩(+칩별 날씨 이모지)
- **Given** — 목록에 gytennis 또는 양평누리 코트가 예약가능 슬롯을 가진 상태
- **When** — 카드가 화면에 표시됨
- **Then** — 예약가능 시간칩이 나열되고, 각 칩에 그 시간대 강수를 나타내는 작은 이모지가 붙는다

성공 기준:
- [ ] 예약가능한 각 시간대가 칩으로 표시된다 (칩 범위는 그 코트의 예약 단위 1h/2h를 반영)
- [ ] 각 칩에 강수 상태 이모지(☀️/🌧️ 등)가 작게 표시되고, 이 이모지 규칙은 다른 소스 카드와 동일하다
- [ ] 예약마감/기간종료 시간대는 칩으로 표시되지 않는다
- [ ] 카드 하단에 "예약하기" 버튼이 활성 상태로 표시된다

---

### 7. 오늘 — 지난 시간대 숨김
- **Given** — 선택 날짜에 오늘이 포함되고 현재 시각이 오후 3시, 오늘 상세를 보는 중
- **When** — 슬롯 소스 카드의 오늘 상세가 표시됨
- **Then** — 현재 시각 이후 시간대만 칩으로 표시된다

성공 기준:
- [ ] 현재 시각(오후 3시) 이전 시간대(06-08, 10-12 등) 칩이 표시되지 않는다
- [ ] 현재 시각 이후 예약가능 시간대만 칩으로 표시된다
- [ ] 현재 시각 이후 남은 예약가능 슬롯이 없으면 "오늘 예약가능 시간대 없음"이 표시된다

---

### 8. 날짜 소스 카드 — yeyak "N면 남음"
- **Given** — 목록에 yeyak 코트가 표시되고 현황 조회에 성공한 상태
- **When** — 카드가 화면에 표시됨
- **Then** — 선택 날짜의 잔여 면수("N면 남음")와 날짜별 날씨 이모지(슬롯 소스와 동일 규칙), "예약하기" 버튼이 표시된다

성공 기준:
- [ ] "N면 남음" 텍스트가 표시된다 (N = 총정원 − 예약수)
- [ ] 날짜별 날씨 이모지가 슬롯 소스 카드와 동일한 규칙으로 표시된다
- [ ] 예약가능 시간칩은 표시되지 않고, "예약 시간대는 사이트에서" 안내가 있다 (yeyak은 날짜 단위)
- [ ] "예약하기" 버튼이 활성 상태로 표시된다

---

### 9. 예약하기 — 슬롯 소스 딥링크
- **Given** — 예약가능 슬롯이 있는 gytennis/양평누리 코트 카드
- **When** — "예약하기" 버튼 클릭
- **Then** — 해당 예약 시스템의 코트·날짜 페이지가 새 탭으로 열린다

성공 기준:
- [ ] 새 탭이 열리고 URL이 해당 소스의 코트·날짜 페이지 형태다 (gytennis: `/rsvDaily/{코트ID}/{날짜}`, 양평누리: 테니스장 대관신청 페이지)
- [ ] 현재 탭(앱)은 그대로 유지된다

---

### 10. 예약하러 가기 — yeyak 딥링크
- **Given** — yeyak 코트 카드
- **When** — "예약하러 가기" 버튼(또는 링크) 클릭
- **Then** — yeyak 예약 상세 페이지가 새 탭으로 열린다

성공 기준:
- [ ] 새 탭이 열리고 URL이 `https://yeyak.seoul.go.kr/web/reservation/selectReservView.do?rsv_svc_id={id}` 형태다
- [ ] 현재 탭(앱)은 그대로 유지된다

---

### 11. 슬롯 소스 예약마감 (남은 슬롯 0)
- **Given** — 선택 날짜에 gytennis/양평누리 코트의 (오늘 탭이면 현재 시각 이후) 예약가능 슬롯이 하나도 없는 상태
- **When** — 카드가 표시됨
- **Then** — "예약마감" 표시가 나타나고 "예약하기" 버튼은 비활성이다

성공 기준:
- [ ] 시간대 칩이 표시되지 않고 "예약마감" 표시가 나타난다
- [ ] "예약하기" 버튼이 비활성(클릭 불가) 상태다

---

### 12. yeyak 예약마감 (0면 남음)
- **Given** — 선택 날짜에 yeyak 코트의 잔여 면수가 0인 상태
- **When** — 카드가 표시됨
- **Then** — "예약마감" 표시가 나타나되, 다른 날짜 확인을 위한 "사이트에서 확인하기" 링크는 활성이다

성공 기준:
- [ ] "예약마감"(또는 "0면 남음") 표시가 나타난다
- [ ] "사이트에서 확인하기" 링크는 클릭 가능 상태로 새 탭을 연다

---

### 13. 코트 상세 정보 표시
- **Given** — 코트 카드가 목록에 표시된 상태
- **When** — 사용자가 해당 코트의 정보를 확인 (정확한 배치 — 인라인 vs 펼침/상세 — 는 wireframe에서 결정)
- **Then** — 소스에서 제공되는 코트 정보(주소·면 수·표면·이용료·운영시간·문의처)가 표시된다

성공 기준:
- [ ] 코트 주소가 표시된다
- [ ] 코트 면 수(예: "4면", "18면")가 표시된다
- [ ] 제공되는 경우 표면 종류·이용료·운영시간·문의처가 표시된다
- [ ] 특정 소스가 제공하지 않는 항목은 표시하지 않는다 (빈 값/미상 표기 없이 생략)

---

### 14. 현황 로드 실패
- **Given** — 앱이 한 소스에서 현황 데이터를 가져오는 중
- **When** — 네트워크 오류 또는 파싱 실패 발생
- **Then** — 해당 코트 카드는 목록에 남고, 예약 현황 대신 "현황 확인 불가" 표시와 활성 외부 링크가 표시된다

성공 기준:
- [ ] 현황 로드 실패한 코트에 "현황 확인 불가" 문구가 표시된다
- [ ] 시간대 칩·"N면 남음"·마감 표시가 나타나지 않는다
- [ ] "사이트에서 직접 확인하기" 링크 버튼은 클릭 가능 상태다
- [ ] 날씨 정보는 (조회 성공 시) 그대로 표시된다

## 불변 규칙

**외부 링크는 항상 새 탭으로 열린다**
— 예약 중에도 코트 목록 컨텍스트를 잃지 않도록, 모든 외부 예약 시스템 링크는 새 탭으로 연다.

**날씨는 코트 위치 좌표 기준으로 조회한다**
— 사용자 위치가 아닌 각 코트의 좌표 기준. 코트와 사용자가 반경 끝까지 떨어질 수 있기 때문.

**날씨 표시는 예약 시스템과 무관하게 완전히 동일하다**
— 날씨는 코트 좌표 기반이므로 세 시스템(gytennis·양평누리·yeyak) 카드에서 동일한 이모지/색상 규칙으로 표시한다(정확한 %/mm 대신 단계별 이모지·색상으로 식별). 예약 시스템에 따라 달라지는 것은 오직 불가피한 예약 데이터뿐이다: (1) 예약 표시 방식(슬롯 소스=시간칩+칩별 이모지 / yeyak=N면 남음+날짜별 이모지), (2) 슬롯 길이(코트·요일별 1h/2h 등). 그 외 모든 표시 요소(날씨 규칙·버튼·레이아웃·코트 정보)는 소스와 무관하게 통일한다.

## 의존성
- **gytennis.or.kr** `/daily/{id}/{date}` (date=YYYY-MM-DD) — 비로그인, 코트면(1~4)×시간대(2h, 06~22시) 슬롯 그리드. 셀 상태는 아이콘 유무로 판별 (2026-07-09 확인). 코트 10개
- **양평누리** (영등포시설공단 `srent.y-sisul.or.kr`) — 비로그인, 하루 8슬롯(2h) `예약가능(n)/예약완료/기간종료` 텍스트 (2026-07-09 확인)
- **yeyak.seoul.go.kr** 코트 상세 — 비로그인, 날짜별 `X/Y`(예약수/총정원) → 잔여 = Y−X (2026-07-09 확인). 시간대별은 로그인 후에만 노출, MVP 미사용
- **시간대별 날씨 API** (기상청 단기예보 또는 대체) — 코트 좌표 기준 오늘/내일/모레의 시간대별 비/눈·강수확률·강수량
- 각 코트의 위도/경도 좌표, gytennis 코트ID, yeyak rsv_svc_id, 양평누리 페이지 URL 사전 등록
- **코트 예약 단위(슬롯 길이)는 코트·요일별로 다름** (예: 목동 평일 1h/주말 2h, gytennis·양평누리 2h). 카드 시간칩은 각 코트의 실제 단위로 렌더링하고, 시간 필터는 1시간 해상도로 겹침 판정한다

## 미결정 항목
- yeyak 대상 코트 범위 및 rsv_svc_id 목록 수집 방법 (정적 등록 vs 검색 파싱)
- 코트 상세 정보의 소스별 제공 범위 편차: 소스마다 노출 항목이 달라(예: yeyak은 이용료·정원·문의처, gytennis는 주소 중심) 어떤 항목을 어디서 채울지 (일부는 사전 수동 등록 가능)
- yeyak 코트의 시간 블록 세분화 여부: 시설이 주간/야간(또는 오전/오후)을 별도 예약항목으로 등록했는지는 시설마다 다름 → rsv_svc_id 수집 시 항목별 확인 필요. 분리 안 된 시설은 시간 필터에서 "시간 미상"으로만 참여
- 시간대별 날씨 매핑: 예보 제공 간격(3시간 등)을 2시간 슬롯에 어떻게 대응시킬지 (plan 단계 확인)
- 슬롯 소스에서 시간대 칩 "예약가능" 집계 기준: gytennis는 코트면별이라, 한 시간대에 ≥1면 가능이면 칩 노출로 가정 (사용자 확인 필요)
