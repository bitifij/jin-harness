# 테니스 레이더 (Tennis Radar)

## Problem Statement
"오늘 테니스 칠 수 있을까?"를 알기 위해 날씨 앱, yeyak.seoul.go.kr,
gytennis.or.kr을 따로따로 뒤져야 하는 마찰을 단 하나의 판단 화면으로 없앤다.

## Recommended Direction
세 예약 시스템(고양시협회 + 영등포시설공단 + 서울시 공공예약)의 현황을
위치 기반으로 통합해 날씨와 함께 보여주고, 예약 가능한 코트는 딥링크로 바로 연결.

데이터 해상도가 소스별로 다르므로 **혼합 해상도(mixed-resolution)** 를 정직하게 표시한다:
- **슬롯 단위** (gytennis, 양평누리): "18:00~20:00 예약가능"
- **날짜 단위** (yeyak): "오늘 3면 남음" (잔여 = 총정원 − 예약수)

## Target Systems
- **gytennis.or.kr**: 고양시 공공코트 10개, 로그인 없이 현황 조회 가능 ✅ 검증완료
  - 현황: GET `/daily/{1~10}/{date}` (date=YYYY-MM-DD). 코트 10개:
    대화·삼송유수지·성라·성사실외·성사전천후·중산·충장·킨텍스유수지·토당·화정
  - 페이지 구성: (1) 월 달력 = 날짜별 `가능 N / 마감 N` 요약,
    (2) 슬롯 그리드 = 코트(1~4) × 시간대(2h, 06~22시 8칸)
  - **셀 상태는 아이콘 유무(CSS)로 표현** — 초록 아이콘=예약가능, 빈칸=마감.
    파서는 텍스트가 아니라 셀 내부 요소를 판별해야 함
  - 예약 실행은 로그인 필요 → 딥링크로 위임
- **양평누리체육공원**: 영등포구시설관리공단 (`srent.y-sisul.or.kr`) ✅ 검증완료
  - 대관신청 `/page/rent/s04.od.list.asp` — 비로그인으로 한 달치 슬롯 전부 노출
  - 슬롯: 2시간 단위, 하루 8칸 (06:00~22:00)
  - 상태 3종: `예약가능(n)` / `예약완료` / `기간종료(n)` (n = 잔여 면수)
  - 구형 ASP, 정적 HTML 테이블 → 파싱 용이

- **yeyak.seoul.go.kr** (서울시): 날짜 단위(reduced tier)로 포함 가능 ✅ 검증완료
  - 딥링크 `/web/reservation/selectReservView.do?rsv_svc_id={id}`
  - 비로그인 노출: 시설정보 + **날짜별 모집현황 `X/Y`** (Y=하루 총정원, X=예약된 수,
    잔여=Y−X). 예: `6/7` = 6면 마감·1면 남음
  - 시간대(슬롯) 단위 및 예약 실행은 로그인 뒤 → 딥링크로 위임
  - 주의: 공개 OpenAPI(data.go.kr)는 시설 상태만 제공, 날짜별 X/Y는 없음 → HTML 파싱

### 제외
- **목동테니스장** (양천구시설관리공단 `ycs.or.kr`, 하드 18면): 대관신청이 로그인
  페이지(`/fmcs/4`)로 직행 → 슬롯 조회가 로그인 뒤. yeyak과 동일 문제로 제외
  (참고: ycs.or.kr 상단에 기온·강수·미세먼지 위젯 상시 노출 — 날씨 소스 힌트)

## Key Assumptions to Validate

      (코트×시간 그리드, 상태는 아이콘 유무로 판별)
- [x] 양평누리(srent.y-sisul.or.kr) 비로그인 슬롯 조회 가능 → 확인됨
- [x] 목동테니스장(ycs.or.kr) → 로그인 필요, 제외
- [ ] 날씨 API로 코트별 위치의 비/눈 예보 조회 가능 (기상청 단기예보)

## MVP Scope
- 위치 기반 반경 설정 (기본 10km)
- 3개 소스 통합: gytennis 10개 코트 + 양평누리 + yeyak 서울 테니스장
- 코트별: 오늘/내일 날씨(☀️🌧️❄️) + 예약 현황
  - 슬롯 소스(gytennis, 양평누리): 시간대별 예약가능 표시
  - 날짜 소스(yeyak): 잔여 면수(N면 남음) 표시
- 예약 버튼 → 딥링크로 해당 코트 예약 페이지 이동
- 날짜 선택 (오늘 / 내일 / 모레)

## Not Doing (and Why)
- 앱 내 직접 예약/결제 — 두 시스템의 인증 체계가 달라 통합 불가 수준
- 민간코트 추가 — 각자 시스템 달라 MVP 범위 초과
- 파트너 매칭 — 다른 문제
- 알림 서비스 — Phase 2, 핵심 먼저 검증 후

- 코트별 위도/경도 좌표 수동 등록 필요 (gytennis 10 + 양평누리 + yeyak 대상)
- (해결) yeyak 공개 API는 날짜별 X/Y 미제공 → HTML 파싱으로 확정
